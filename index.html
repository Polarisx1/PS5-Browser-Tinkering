III. Field kit

working tools built on confirmed primitives
The field kit treats the PS5 browser as a real destination, not just a subject of study. Each tool below combines confirmed-working primitives (Cache API, crypto, fetch, localStorage) into something useful you can actually run on a PS5 today.
K-01 // PICKUP PAD
End-to-end encrypted message channel, PC → PS5
A keypair is generated on the PS5. Senders encrypt with the public key; only the PS5 can decrypt. Free relay carries the bytes blindly.
CRYPTO + RELAY

This PS5 — receiver

Your pickup code is derived from a keypair stored locally in this browser. Share the code to receive notes.

Loading keypair…
— — — —
How this works

A fresh ECDH P-256 keypair is generated and persisted in localStorage. The first 6 chars of the SHA-256 hash of the public key form your pickup code. Senders fetch the public key via that code, derive a shared AES-GCM key, encrypt the message, and post the ciphertext + ephemeral public key to the relay. Only your PS5 can derive the shared secret to decrypt. The relay (jsonbin.io anonymous bin or similar free service) only ever sees opaque ciphertext.

Sender (from any PC/phone)

Drop in the receiver's pickup code, write a note, hit SEND. Encrypts client-side before posting.

This is the headline demonstration. A console-resident program with no installed software receives end-to-end encrypted messages from anywhere on the internet, with only the browser, free public infrastructure, and standards-compliant crypto. The relay cannot read messages. Sony cannot read messages. No third party can read messages.
